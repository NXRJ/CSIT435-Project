# Experimental People and General-Object Detector

## Purpose and project alignment

This experiment adds a parallel people/general-object branch while preserving the project's custom-trained RBF SVM for accessibility markers. The new branch is optional and appears in a separate Gradio tab when `CSCI435_ENABLE_GENERAL=1`. It does not replace the original enhancement, edge, marker recognition, motion, or tracking pipeline.

## Dataset audit

The supplied folder `C:\Users\PC\Downloads\train\train` contained 240 readable JPEG images but no class labels, bounding boxes, CSV, XML, COCO JSON, or YOLO text annotations. Therefore, supervised detector training and genuine ground-truth accuracy measurement were not possible from the folder as supplied.

For an exploratory result, pretrained YOLO11n generated pseudo-labels at confidence >= 0.35. The deterministic split contained 192 training and 48 validation images. It produced 1,016 boxes across 194 labelled images, including 592 person boxes. The mean accepted pseudo-label confidence was 64.6%.

## Training configuration

- Base model: pretrained YOLO11n with 80 COCO classes.
- Image size: 512 x 512.
- Batch size: 8.
- Requested epochs: 6 on CPU; early stopping selected the best checkpoint from epoch 1 after the run ended at epoch 4.
- First ten backbone layers frozen.
- Deterministic seed: 435.
- Output: `artifacts/general_object_training/general_objects_pseudo_finetuned.pt`.

## Results

All values below are measured against labels generated by the baseline itself. They measure pseudo-label consistency, not independent real-world accuracy.

| Measure | Pretrained baseline | Fine-tuned | Change |
|---|---:|---:|---:|
| mAP@50 | 86.96% | 87.64% | +0.68 percentage points |
| mAP@50-95 | 79.98% | 81.55% | +1.57 percentage points |
| Precision | 65.41% | 60.16% | -5.25 percentage points |
| Recall | 86.13% | 81.98% | -4.16 percentage points |
| CPU latency | 22.43 ms/image | 20.42 ms/image | -2.01 ms |
| CPU throughput | 44.58 FPS | 48.97 FPS | +4.38 FPS |

Person-specific pseudo-label results changed from 85.56% precision, 97.98% recall and 91.15% mAP@50-95 to 78.48% precision, 98.99% recall and 90.48% mAP@50-95. The fine-tune found slightly more people but produced more false positives at the selected operating point.

The fully integrated marker-SVM plus general-object path was also benchmarked over 20 warmed runs of a dark multi-person image. Mean wall-clock latency was 37.97 ms, 95th-percentile latency was 39.07 ms, and throughput was 26.34 FPS. This remains above the project's 10 FPS requirement even when both models run sequentially.

## Qualitative findings

- A single outdoor person was detected at 92% confidence with a well-localised box.
- In a dark indoor scene, the model detected four people plus a dining table, bottle, and wine glasses.
- On the project's generated marker scene, it detected the stop marker as a COCO stop sign but misclassified the green safe marker as a sports ball and missed the warning marker.

The last result is why the original marker SVM remains necessary. The two models provide complementary capabilities: the SVM handles project-specific marker semantics, while YOLO adds broader people/object awareness.

## Conclusion

The experiment demonstrates that the supplied images can support self-training, but the improvement is small and the precision/recall trade-off is mixed. The fine-tuned weights are suitable only as an optional experimental branch. A defensible final general-object model requires manually reviewed bounding boxes and an independently annotated test split. Until those labels exist, the original rubric results must remain the primary reported results.

## Pretrained model reference

The experiment starts from Ultralytics YOLO11n (`yolo11n.pt`), pretrained on the 80-class COCO detection task. It is an optional, clearly identified pretrained component; the rubric-critical custom model remains the group's RBF SVM. See the [official YOLO11 documentation](https://docs.ultralytics.com/models/yolo11/) and [Ultralytics repository](https://github.com/ultralytics/ultralytics).
